# 構成図 — 要件定義 1「データ収集」

## システム構成図

```mermaid
flowchart TD
    User(["👤 ユーザー入力\n開始日・終了日・店舗URL"])

    subgraph メイン["メイン.py（エントリーポイント）"]
        Input["入力バリデーション\n・日付形式チェック\n・URL形式チェック"]
        SlugExtract["extract_slug()\nURLからホール名(slug)抽出"]
        DateFilter["処理済み日付スキップ\n(get_processed_dates)"]
        Loop["日付ループ\n未取得日付のみ処理"]
        RateLimit["レート制御\n・通常待機: max(10s, 40s-経過時間)\n・バッチ休憩: 5分/20件ごと"]
        ErrorHandle["エラーハンドリング\n・403 → 即時中止\n・その他 → ログ記録して続行"]
    end

    subgraph Scraper["scraper.py（取得・解析）"]
        Session["create_session()\nUser-Agent / Referer 等ヘッダー設定"]
        BuildURL["build_url()\nhttps://ana-slo.com/{日付}-{slug}-data/"]
        FetchPage["fetch_page()\nHTTP GET\nリトライ最大3回\n指数バックオフ(60s, 120s, 240s)"]
        Parse["get_info()\nBeautifulSoupでHTML解析\n機種ごとにsectionを走査\nカラム数を自動検出"]
    end

    subgraph WebSite["外部サイト"]
        AnaSlo["🌐 ana-slo.com\nパチスロデータサイト"]
    end

    subgraph DB["db.py（保存・管理）"]
        SetupDB["setup_db()\nテーブル作成 or スキーマmigration\n(旧スキーマは slot_data_old に退避)"]
        GetDates["get_processed_dates()\nSELECT DISTINCT 日付"]
        WriteDB["write_db()\nINSERT OR IGNORE → slot_data\n数値変換(_to_int / _to_prob)"]
        WriteMissing["write_missing()\nINSERT → missing_data"]
        WriteNull["write_null_record()\n機種名のみ判明した欠損を\nslot_data にNULLプレースホルダー挿入"]
    end

    subgraph Storage["ホールデータ/ (SQLite)"]
        DBFile["{ホール名}.db"]
        subgraph Tables["テーブル"]
            SlotData["slot_data\n日付/ホール名/機種名/台番号\n回転数/差枚/BB/RB/ART\nBB確率/RB確率/ART確率/合成確率\nUNIQUE(日付, ホール名, 機種名, 台番号)"]
            MissingData["missing_data\n日付/ホール名/機種名\n理由/記録日時"]
        end
    end

    %% メインフロー
    User --> Input
    Input --> SlugExtract
    SlugExtract --> SetupDB
    SetupDB --> GetDates
    GetDates --> DateFilter
    DateFilter --> Loop

    Loop --> BuildURL
    BuildURL --> Session
    Session --> FetchPage
    FetchPage -->|"HTTP GET"| AnaSlo
    AnaSlo -->|"HTMLレスポンス"| FetchPage
    FetchPage --> Parse

    Parse -->|"data_list\ndata_column_list\ndata_row_list"| WriteDB
    Parse -->|"missing_machines\n(機種名, 理由)"| WriteMissing
    Parse -->|"機種名判明の欠損"| WriteNull

    WriteDB --> SlotData
    WriteMissing --> MissingData
    WriteNull --> SlotData

    Loop --> RateLimit
    RateLimit -->|"次の日付へ"| Loop

    Parse -.->|"403エラー"| ErrorHandle
    ErrorHandle -.->|"即時終了"| メイン

    SlotData --> DBFile
    MissingData --> DBFile
```

## 処理フロー補足

| フェーズ | 処理 | 担当モジュール |
|---|---|---|
| ① 入力受付 | 日付・URL検証、slug抽出 | `メイン.py` + `scraper.extract_slug` |
| ② DB初期化 | テーブル作成・スキーマmigration | `db.setup_db` |
| ③ 重複スキップ | 取得済み日付をDBから取得して除外 | `db.get_processed_dates` |
| ④ URL構築 | `https://ana-slo.com/{日付}-{slug}-data/` | `scraper.build_url` |
| ⑤ HTTP取得 | リトライ3回・指数バックオフ・SSL対応 | `scraper.fetch_page` |
| ⑥ HTML解析 | section単位でカラム数自動検出・台データ抽出 | `scraper.get_info` |
| ⑦ データ保存 | 正常データ → `slot_data`、欠損 → `missing_data` | `db.write_db` / `write_missing` / `write_null_record` |
| ⑧ レート制御 | 通常10〜40秒待機、20件ごとに5分休憩 | `メイン.py` |

## DBスキーマ詳細

### slot_data テーブル

| カラム | 型 | 備考 |
|---|---|---|
| 日付 | TEXT | YYYY-MM-DD |
| ホール名 | TEXT | URLスラッグ |
| 機種名 | TEXT | |
| 台番号 | INTEGER | |
| 回転数 | INTEGER | カンマ除去して保存 |
| 差枚 | INTEGER | 符号付き（+/−）・カンマ除去 |
| BB / RB / ART | INTEGER | ART非搭載機種はNULL |
| BB確率 / RB確率 / ART確率 / 合成確率 | REAL | `1/xxx` → `1÷xxx` の実数。分母0はNULL |

- UNIQUE制約: `(日付, ホール名, 機種名, 台番号)`

### missing_data テーブル

| カラム | 型 | 備考 |
|---|---|---|
| 日付 | TEXT | |
| ホール名 | TEXT | |
| 機種名 | TEXT | ホール全体欠損時はNULL |
| 理由 | TEXT | `'ページにデータなし'` / `'カラム数特定不可'` |
| 記録日時 | TEXT | 自動付与 |
